# Интеллектуальный агент для абитуриентов (LLM + RAG)

Telegram-бот для ответов на вопросы абитуриентов о поступлении, образовательных программах, стоимости обучения, проходных баллах, количестве мест и требованиях к ЕГЭ.

Проект принимает вопрос на естественном языке, ищет релевантную информацию в базе знаний университета и формирует понятный ответ с использованием комбинации:

- **FAQ-поиска по эмбеддингам**
- **RAG по базе знаний**
- **SQL-агента для табличных данных по образовательным программам**
- **LLM для генерации итогового ответа**
- **Telegram-интерфейса с inline-кнопками**

> Текущая реализация ориентирована на данные Президентской академии (РАНХиГС), но архитектура легко адаптируется под другой вуз при замене источников данных.

---

## Что умеет система

Бот отвечает на типовые вопросы абитуриентов, например:

- какие ЕГЭ нужны для поступления на программу;
- сколько стоит обучение;
- какой проходной балл был в прошлом году;
- сколько бюджетных и платных мест доступно;
- какие документы нужны для поступления;
- чем отличаются программы, мегакластеры и направления;
- какие программы подходят под заданные баллы ЕГЭ.

Дополнительно реализованы:

- фильтрация токсичных сообщений;
- подсказка похожего FAQ-вопроса, если совпадение неидеальное;
- прямой доступ к карточке найденной программы через кнопки в Telegram;
- fallback-режим без LLM, в котором бот всё равно возвращает найденные данные.

## Архитектура решения

Система использует гибридный пайплайн обработки вопроса:

```text
Пользователь -> Telegram Bot -> AssistantPipeline
                         |
                         +-> ToxicityFilter
                         +-> FAQ retrieval (embeddings + FAISS)
                         +-> Program search (SQLite + fuzzy/full-text heuristics)
                         +-> Knowledge Base retrieval (RAG)
                         +-> Query classification
                         |      |
                         |      +-> SQL Agent for program data
                         |      +-> RAG answer for general questions
                         |
                         +-> LLM -> финальный ответ пользователю
```

### Логика обработки запроса

1. **Проверка токсичности**
   - сообщение проходит через словарь стоп-слов и ML-классификатор токсичности;
   - токсичные запросы не попадают в основной пайплайн.

2. **Поиск ответа в FAQ**
   - вопрос пользователя очищается и переводится в эмбеддинг;
   - система ищет ближайшие вопросы в FAQ через **FAISS**;
   - при высоком сходстве сразу возвращается готовый ответ;
   - при среднем сходстве бот предлагает пользователю подтвердить похожий вопрос.

3. **Маршрутизация запроса**
   - если вопрос связан с конкретными программами, стоимостью, баллами, местами или ЕГЭ — используется **SQL-ветка**;
   - если вопрос общий (правила приёма, документы, структура обучения и т.д.) — используется **RAG-ветка**.

4. **Формирование ответа**
   - LLM получает только релевантный контекст;
   - в промптах жёстко задано ограничение: **не выдумывать факты и отвечать только на основе найденных данных**.

## Технологический стек

- **Python 3.10+**
- **pyTelegramBotAPI** — Telegram-бот
- **pandas / openpyxl** — загрузка Excel-файлов
- **SQLite** — локальная база данных
- **sentence-transformers** — эмбеддинги для FAQ и базы знаний
- **FAISS** — быстрый nearest-neighbor поиск
- **OpenAI-compatible API / Mistral** — генерация ответа LLM
- **thefuzz** — нечёткий поиск программ
- **transformers + torch** — токсик-фильтр
- **Natasha** — морфологическая обработка русского языка
- **CrossEncoder** — дополнительный семантический слой в пайплайне

Файлы:

## Как это работает внутри

### FAQ-поиск

Для FAQ строится векторный индекс вопросов. Если новый запрос очень похож на один из уже известных вопросов, система возвращает готовый ответ без генерации.

Это даёт:

- быстрый ответ;
- стабильность на типовых запросах;
- меньше галлюцинаций;
- меньшую нагрузку на LLM.

### RAG по базе знаний

Для общих вопросов бот ищет релевантные фрагменты в базе знаний и передаёт их в LLM как контекст.

Это полезно для вопросов вида:

- «Какие документы нужны для поступления?»
- «Что такое мегакластер?»
- «Как устроено обучение?»

### SQL-агент по образовательным программам

Для вопросов о программах, стоимости, баллах, местах и ЕГЭ используется отдельная ветка:

1. LLM генерирует безопасный `SELECT` для SQLite.
2. Запрос проходит валидацию (`INSERT/UPDATE/DELETE/...` запрещены).
3. Полученные строки форматируются в ответ пользователю.

Это особенно хорошо работает на запросах:

- «Сколько стоит бизнес-информатика?»
- «Какие ЕГЭ нужны для анализа данных?»
- «Где больше всего бюджетных мест?»
- «Какая самая дорогая программа?»

### Поиск программ и подбор по ЕГЭ

В `program_search.py` реализованы:

- точное и частичное совпадение по названию;
- нечёткий поиск;
- полнотекстовый поиск по описанию, навыкам и мегакластеру;
- нормализация названий предметов ЕГЭ;
- подбор подходящих программ по введённым баллам.

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <repo_url>
cd abiturient-bot-main
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

> В коде также используется пакет `natasha`. Если он не установлен через `requirements.txt`, добавьте его вручную:

```bash
pip install natasha
```

### 4. Подготовить `.env`

Создайте файл `.env` в корне проекта:

```env
TELEGRAM_TOKEN=your_telegram_bot_token
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://api.mistral.ai/v1
LLM_MODEL=mistral-small-latest
```

Пояснения:

- `TELEGRAM_TOKEN` — токен Telegram-бота;
- `OPENAI_API_KEY` — ключ для LLM-провайдера с OpenAI-compatible API;
- `OPENAI_BASE_URL` — адрес API (по умолчанию указан Mistral);
- `LLM_MODEL` — имя используемой модели.

### 5. Добавить данные

Создайте папку `data/` и положите туда:

```text
data/
├── Database.xlsx
├── Database-2.xlsx
├── all_program.xlsx
├── ru_abusive_words.txt
└── ru_curse_words.txt
```

### 6. Запустить бота

```bash
python bot.py
```

При первом запуске пайплайн:

- соберёт SQLite-базу `data/ranepa.db`;
- загрузит стоп-слова;
- создаст FAISS-индексы;
- инициализирует Telegram-бота.

## Сильные стороны решения

- **Гибридная архитектура**: FAQ + RAG + SQL работают вместе, а не конкурируют друг с другом.
- **Снижение галлюцинаций**: ответы строятся на найденных данных, а не “из головы”.
- **Удобство для абитуриента**: естественный язык вместо фильтров и сложных форм.
- **Масштабируемость**: можно подключить новые вузы и новые источники данных.
- **Работа даже без LLM**: часть сценариев поддерживается fallback-логикой.

## Ограничения текущей версии

Важно понимать ограничения текущего репозитория:

1. **Данные не входят в архив**  
   Папка `data/` отсутствует в репозитории и должна быть подготовлена отдельно.

2. **Качество ответа зависит от качества Excel-источников**  
   Если данные неполные, устаревшие или по-разному оформлены, ответы будут хуже.

3. **Нет полноценного API или web-интерфейса**  
   На текущий момент основной пользовательский интерфейс — Telegram-бот.

4. **Часть логики жёстко заточена под конкретный вуз**  
   Например, системные промпты и названия сущностей ориентированы на РАНХиГС.

5. **Нужна аккуратная настройка зависимостей**  
   В коде используются библиотеки, которые желательно синхронизировать с `requirements.txt`.

## Коротко о ключевой идее

Главная ценность проекта в том, что он не пытается отвечать на вопросы абитуриентов “в общем”, а опирается на **конкретные университетские данные**:

- FAQ для типовых вопросов;
- база знаний для регламентов и правил;
- таблица образовательных программ для точных числовых ответов;
- LLM — как слой интерпретации и генерации понятного ответа.

Именно это делает систему практичной для реального использования.
